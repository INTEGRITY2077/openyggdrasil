# Read SCHEMA.md, index.md, and recent log.md from the canonical wiki.

## Question

What contract should a vault-aware answer follow after reading `SCHEMA.md`,
`index.md`, and recent `log.md` entries?

## Answer

A vault-aware answer should state the decision, name the supporting vault pages,
explain why the answer is reusable, and identify whether the result should be
filed, updated, or left unpromoted.

The answer should not copy raw provider session text into the vault. It should
use public-safe source refs and update `index.md` or `log.md` only when durable
public memory actually changed.
